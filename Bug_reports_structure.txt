Структура баг репорта.

-- Summary (название) - Что? Где? Когда (при каких обстоятельствах)?
-- Description (описание):
   - Steps to reproduce (STR - шаги по воспроизведению)
   - Actual result (фактический результат)
   - Expected result (ожидаемый результат)
-- Attachment (приложение в виде фото, видео, гиф и др.)
-- Environment (окружение: ОС, версия браузера или приложения, разрешение экрана)
-- Priority (приоритет исправления)
-- Severity (степень влияния на работоспособность продукта)
-- Assignee (на кого назначен)
-- id (уникальный идентификационный номер)
